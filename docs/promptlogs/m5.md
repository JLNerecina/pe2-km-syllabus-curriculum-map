## Entry 2
**Date:** April 28, 2026
**Task:** Fixing Vitest failures in the Map component.

**Prompt given to AI:** "why did this line fail in testing?" (with screenshot of `useNavigate` error).

**What the AI produced:** Explained that `useNavigate` requires a `Router` context and provided a fix using `MemoryRouter`. It also identified a race condition in asynchronous Supabase mocks.

**What I changed / improved:** I wrapped the component in `<MemoryRouter>` and moved all `expect` assertions inside a `waitFor` block.

**Why:** To provide the necessary React Router context and ensure the test waits for the async data chain to finish before evaluating.

**What I learned / decided:** Hooks like `useNavigate` are context-dependent; testing them requires specific wrappers. Asynchronous chains in Vitest need consolidated `waitFor` blocks to prevent premature assertion failures.

## Entry 3
**Date:** May 3, 2026
**Task:** Debugging Audit Log search inconsistencies.

**Prompt given to AI:** "some searches don't perform correctly... searching 'Profile edit' shows nothing."

**What the AI produced:** Identified a "Global Filter" bug where the search only targeted the `Actor` column using strict equality instead of a case-insensitive partial match across all columns.

**What I changed / improved:** Updated the filter logic to use `.toLowerCase()` and `.includes()` across `Actor`, `Action`, and `Target` fields.

**Why:** Users expect a global search that finds keywords regardless of the column or text casing.

**What I learned / decided:** Always normalize strings for search. Relying on strict equality (`===`) for UI search bars leads to poor UX and perceived data loss.
