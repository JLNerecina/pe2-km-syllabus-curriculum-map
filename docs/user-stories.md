# User Stories
**Project:** Syllabus & Curriculum Map (Knowledge Mapping)  
**Development Approach:** AI-assisted coding using Google AI Studio (Gemini). Developer prompts, reviews, refines, and owns all generated code.  
**Date:** 2026-04-02  
**Total Points:** 65

### Full Product Backlog

| ID      | User Story                                                                 | Story Points | Priority |
|---------|----------------------------------------------------------------------------|--------------|----------|
| US-01   | As a **Super Admin**, I want to manage Admin accounts so that I can control system access | 3            | High     |
| US-02   | As a **Super Admin**, I want to view an audit trail of all changes so that I can monitor knowledge modifications | 5            | High     |
| US-03   | As an **Admin**, I want to manage Faculty and Student accounts so that user data is properly maintained | 5            | High     |
| US-04   | As a **Faculty**, I want to add/edit/delete courses with skills and prerequisites so that the curriculum map can be built | 8            | High     |
| US-05   | As a **Faculty**, I want to create prerequisite relationships between courses so that knowledge flow is correctly visualized | 8            | High     |
| US-06   | As a **Student**, I want a personal dashboard showing my program's curriculum map so that I can view my learning path | 5            | High     |
| US-07   | As any **authenticated user**, I want to view an interactive visual curriculum map (nodes + edges) so that I can understand course connections | 13           | High     |
| US-08   | As any **authenticated user**, I want to search and filter the map by year, semester, or skill so that I can quickly find relevant knowledge | 5            | High     |
| US-09   | As a **Faculty or Student**, I want to click a course node to see detailed skills and prerequisites so that I can explore knowledge dependencies | 5            | Medium   |
| US-10   | As an **Admin**, I want to restore archived courses or users so that data is never permanently lost | 3            | Medium   |
| US-11   | As a **Super Admin**, I want to export the curriculum map as JSON/PDF so that it can be shared | 3            | Low      |
| US-12   | As any **user**, I want the application to be responsive so that I can access it on any device | 5            | Low      |

### Sprint 1:
Developer builds core features. Each member works on their deliverables in parallel on their branch.

**Stories for Sprint 1 (Total: 21 points)**  
- US-01 Super Admin – manage Admin accounts (3)  
- US-02 Super Admin – view audit trail (5)  
- US-03 Admin – manage Faculty & Student accounts (5)  
- US-06 Student – personal dashboard (basic) (5)  
- US-07 (partial) – basic authentication + role-based access (3)

**Focus:** Authentication, role system, and basic dashboards.

### Sprint 2:
Features integrated into dev branch. QA Lead begins testing. Bug reports filed as GitHub Issues.

**Stories for Sprint 2 (Total: 26 points)**  
- US-04 Faculty – add/edit/delete courses with skills (8)  
- US-05 Faculty – create prerequisite relationships (8)  
- US-07 Interactive visual curriculum map (nodes + edges) (5)  
- US-08 Search and filter functionality (5)

**Focus:** Core knowledge mapping features (visual graph, CRUD, relationships).

**Remaining stories (US-09 to US-12)** will be handled in future sprints or polish phase.

**Team Velocity Target:** 20–25 points per sprint (to be measured after Sprint 1).
